<h1>üåà RDW ‚Äî Rainbow Data Winner</h1><p></p><h3><strong>Official Specification ‚Äî Version 1.0</strong></h3><p></p><p>RDW is a structured, safe, compact data ecosystem invented by Rudra.<br>It consists of <strong>three file types</strong>, each with a strict purpose:</p><p></p><p>File TypeExtensionPurpose<strong>DATA RDW</strong><code>.data.rdw</code>Stores structured data<strong>CRUD RDW</strong><code>.crud.rdw</code>Authorized editing instructions<strong>QRY RDW</strong><code>.qry.rdw</code>Query and extraction instructions</p><p>All three formats share the same <strong>RDW-native syntax style</strong>:</p><p></p><ul><li><p><code>{RowName}</code> for rows</p><p></p></li><li><p><code>(ColumnName)</code> for columns</p><p></p></li><li><p><code>Key*Value</code> for headers</p><p></p></li><li><p>Escaped <code>'values'</code> for safety</p><p></p></li><li><p>Structured sections: <code>[Headers]</code>, <code>[Data]</code>, <code>[Operations]</code></p><p></p></li></ul><p></p><p>RDW enforces <strong>strict separation of concerns</strong>:</p><p></p><ul><li><p>DATA files <strong>cannot</strong> be edited directly</p><p></p></li><li><p>CRUD files <strong>cannot</strong> read data</p><p></p></li><li><p>QRY files <strong>cannot</strong> modify data</p><p></p></li></ul><p></p><p>This makes RDW safe, auditable, and predictable.</p><p></p><h1>üì¶ DATA RDW ‚Äî <code>.data.rdw</code></h1><p></p><h3><strong>RDW Data Storage Format</strong></h3><p></p><p>The DATA RDW file is the <strong>canonical storage container</strong> for RDW datasets.</p><p></p><p>It contains two sections:</p><p></p><p>Code</p><pre><code>[Headers]
...
[Data]
...
</code></pre><h2>1. [Headers] Section</h2><p></p><p>Code</p><pre><code>[Headers]
CRUDeditor*/path/to/editor/file.crud.rdw
Compact*yes|no
Encoding*UTF-8|UTF-16|UTF-32
Name*&lt;dataset_name_or_purpose&gt;
</code></pre><h3>Header meanings</h3><p></p><ul><li><p><strong>CRUDeditor</strong>*<br>Path to the <code>.crud.rdw</code> file authorized to modify this dataset.</p><p></p></li><li><p><strong>Compact</strong>*</p><p></p><ul><li><p><code>no</code> ‚Üí human-readable structured text</p><p></p></li><li><p><code>yes</code> ‚Üí <strong>RDW Bytecode Mode</strong></p><p></p><ul><li><p>Converts the entire <code>[Data]</code> section into a <strong>super‚Äëcompact binary format</strong></p><p></p></li><li><p>Only a QRY RDW engine can decode it</p><p></p></li><li><p>CRUD RDW still works because it modifies the <em>logical structure</em>, not the raw bytes</p><p></p></li></ul><p></p></li></ul><p></p></li><li><p><strong>Encoding</strong>*<br>Text encoding for string values.</p><p></p></li><li><p><strong>Name</strong>*<br>Human-readable dataset label.</p><p></p></li></ul><p></p><h2>2. [Data] Section</h2><p></p><p>RDW uses a <strong>vertical, compact, hierarchical layout</strong>:</p><p></p><ul><li><p>Rows declared with <code>{RowName}</code></p><p></p></li><li><p>Columns declared with <code>(ColumnName)</code></p><p></p></li><li><p>Values appear at the intersection</p><p></p></li></ul><p></p><h3>Syntax</h3><p></p><p>Code</p><pre><code class="language-auto">[Data]

{RowName}
    (ColumnA) valueA
    (ColumnB) valueB

{AnotherRow}
    (ColumnA) valueX
</code></pre><h3>Rules</h3><p></p><ul><li><p>A row stays active until another row is declared.</p><p></p></li><li><p>A column applies only to the next value.</p><p></p></li><li><p>Values may be:</p><p></p><ul><li><p>numbers</p><p></p></li><li><p>strings</p><p></p></li><li><p>lists</p><p></p></li><li><p>encoded binary</p><p></p></li></ul><p></p></li><li><p>Indentation is optional.</p><p></p></li></ul><p></p><h2>3. Compact Mode (Bytecode)</h2><p></p><p>If <code>Compact*yes</code>, the <code>[Data]</code> section becomes:</p><p></p><ul><li><p>A compressed bytecode stream</p><p></p></li><li><p>Using:</p><p></p><ul><li><p>dictionary compression</p><p></p></li><li><p>row/column indexing</p><p></p></li><li><p>delta encoding</p><p></p></li><li><p>string table</p><p></p></li></ul><p></p></li><li><p>Only QRY RDW engines can decode it</p><p></p></li><li><p>Human-readable form is not stored</p><p></p></li></ul><p></p><p>This makes RDW extremely efficient for:</p><p></p><ul><li><p>transfer</p><p></p></li><li><p>storage</p><p></p></li><li><p>embedding</p><p></p></li><li><p>large datasets</p><p></p></li></ul><p></p><h1>üõ† CRUD RDW ‚Äî <code>.crud.rdw</code></h1><p></p><h3><strong>RDW Editing Format</strong></h3><p></p><p>A CRUD RDW file is the <strong>only</strong> way to modify a <code>.data.rdw</code> dataset.</p><p></p><p>It contains:</p><p></p><p>Code</p><pre><code>[Headers]
...
[Operations]
...
</code></pre><h2>1. [Headers] Section</h2><p></p><p>Code</p><pre><code>[Headers]
Data*/path/to/data/file.data.rdw
Rows*&lt;row_list_or_all&gt;
Columns*&lt;column_list_or_all&gt;
</code></pre><h3>Header meanings</h3><p></p><ul><li><p><strong>Data</strong>*<br>Path to the DATA RDW file this CRUD file modifies.</p><p></p></li><li><p><strong>Rows</strong>*<br>Which rows this CRUD file is allowed to modify.</p><p></p></li><li><p><strong>Columns</strong>*<br>Which columns this CRUD file is allowed to modify.</p><p></p></li></ul><p></p><p>Unauthorized edits are invalid.</p><p></p><h2>2. [Operations] Section</h2><p></p><p>CRUD supports <strong>three operation types</strong>:</p><p></p><ul><li><p><strong>edit</strong> ‚Äî change an existing value</p><p></p></li><li><p><strong>add</strong> ‚Äî add a new column/value to an existing row</p><p></p></li><li><p><strong>remove</strong> ‚Äî remove a column/value</p><p></p></li></ul><p></p><h2>EDIT Operation</h2><p></p><p>Code</p><pre><code>edit = {RowName}, (ColumnName) ...
        replace = 'old_value' ...
        with = 'new_value'
</code></pre><p>Rules:</p><p></p><ul><li><p>Row must exist</p><p></p></li><li><p>Column must exist</p><p></p></li><li><p>Old value must match exactly</p><p></p></li><li><p>New value replaces it</p><p></p></li></ul><p></p><h2>ADD Operation</h2><p></p><p>Code</p><pre><code>add = {RowName}, (ColumnName) ...
       add = 'value_to_add'
</code></pre><p>Rules:</p><p></p><ul><li><p>Row must exist</p><p></p></li><li><p>Column must NOT exist</p><p></p></li><li><p>Adds a new column/value pair</p><p></p></li></ul><p></p><h2>REMOVE Operation</h2><p></p><p>Code</p><pre><code>remove = {RowName}, (ColumnName)
</code></pre><p>Rules:</p><p></p><ul><li><p>Row must exist</p><p></p></li><li><p>Column must exist</p><p></p></li><li><p>Removes the entire column/value pair</p><p></p></li></ul><p></p><h1>üîç QRY RDW ‚Äî <code>.qry.rdw</code></h1><p></p><h3><strong>RDW Query Format</strong></h3><p></p><p>A QRY RDW file is the <strong>only</strong> way to read, search, filter, or extract data from a <code>.data.rdw</code> file.</p><p></p><p>It contains:</p><p></p><p>Code</p><pre><code>[Headers]
...
[Operations]
...
</code></pre><h2>1. [Headers] Section</h2><p></p><p>Code</p><pre><code>[Headers]
DataFile*/path/to/data/file.data.rdw
Name*&lt;query_name&gt;
</code></pre><ul><li><p><strong>DataFile</strong>* ‚Äî dataset to query</p><p></p></li><li><p><strong>Name</strong>* ‚Äî human-readable query name</p><p></p></li></ul><p></p><h2>2. [Operations] Section</h2><p></p><p>All query values must be <strong>escaped</strong> and wrapped in <code>'...'</code> for safety.</p><p></p><p>RDW supports the following operations:</p><p></p><h1>‚úî FIND</h1><p></p><p>Search for keywords, keyphrases, row names, or column names.</p><p></p><p>Code</p><pre><code>find = {optional_rows}, (optional_columns) ...
       key = 'search_term' ...
       give = 'output_target'
</code></pre><h1>‚úî GET</h1><p></p><p>Retrieve a specific row/column intersection.</p><p></p><p>Code</p><pre><code>get = {RowName}, (ColumnName) ...
      give = 'output_target'
</code></pre><h1>‚úî LIST</h1><p></p><p>List rows, columns, or both.</p><p></p><p>Code</p><pre><code>list = {rows} ... give = 'console'
list = (columns) ... give = 'console'
list = {rows}, (columns) ... give = 'console'
</code></pre><h1>‚úî FILTER</h1><p></p><p>Filter rows based on column conditions.</p><p></p><p>Code</p><pre><code>filter = {rows}, (column) ...
         where = 'condition' ...
         give = 'output_target'
</code></pre><h1>‚úî COUNT</h1><p></p><p>Count rows or columns matching a condition.</p><p></p><p>Code</p><pre><code>count = {rows}, (columns) ...
        where = 'condition' ...
        give = 'output_target'
</code></pre><h1>‚úî EXISTS</h1><p></p><p>Check if a row or column exists.</p><p></p><p>Code</p><pre><code>exists = {RowName} ... give = 'console'
exists = (ColumnName) ... give = 'console'
</code></pre><h1>‚úî DUMP</h1><p></p><p>Dump the entire dataset or a subset.</p><p></p><p>Code</p><pre><code>dump = {optional_rows}, (optional_columns) ...
       give = 'output_target'
</code></pre><h1>‚úî RANGE</h1><p></p><p>Query numeric ranges.</p><p></p><p>Code</p><pre><code>range = {rows}, (column) ...
        from = 'min' ...
        to = 'max' ...
        give = 'output_target'
</code></pre><h1>‚úî SEARCH-STRUCTURE</h1><p></p><p>Search row/column names.</p><p></p><p>Code</p><pre><code>search-structure = 'query' ...
                   give = 'console'
</code></pre><h1>‚úî SCRIPT</h1><p></p><p>Run a Lua script on the results.</p><p></p><p>Code</p><pre><code>script = 'lua_code_here'
</code></pre><h1>üåà RDW ECOSYSTEM SUMMARY</h1><p></p><p>ComponentPurposeEditable?Readable?<strong>DATA RDW</strong>Stores structured data‚ùå No‚úî Yes (via QRY)<strong>CRUD RDW</strong>Modifies DATA RDW‚úî Yes‚ùå No<strong>QRY RDW</strong>Reads DATA RDW‚ùå No‚úî Yes</p><p>RDW is:</p><p></p><ul><li><p>Compact</p><p></p></li><li><p>Safe</p><p></p></li><li><p>Structured</p><p></p></li><li><p>Extensible</p><p></p></li><li><p>Human-readable (unless Compact*yes)</p><p></p></li><li><p>Machine-optimized</p><p></p></li></ul><p></p>
